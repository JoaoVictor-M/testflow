Guia de Configuração de Email

Este documento orienta como configurar o servidor SMTP no TestFlow para envio de credenciais e notificações de sistema.

Acesso

  1. Faça login como Administrador.
  2. Acesse o menu Gerenciamento > Configurações de Email.

Exemplos de Configuração

1. Gmail (Google Workspace ou Pessoal)

Importante: Contas pessoais requerem o uso de "Senha de Aplicativo" (App Password) devido à autenticação de dois fatores (2FA). Não use sua senha de login normal.
  • Host SMTP: smtp.gmail.com
  • Porta: 465
  • Usuário: seu.email@gmail.com
  • Senha: xxxx xxxx xxxx xxxx (Sua Senha de App de 16 dígitos)
  • Conexão Segura (SSL/TLS): Marcado (Sim)
  • Email de Envio (From): TestFlow <seu.email@gmail.com>

2. Microsoft Outlook / Office 365

Para contas Office 365, o SMTP AUTH deve estar habilitado na organização.
  • Host SMTP: smtp.office365.com
  • Porta: 587
  • Usuário: seu.email@empresa.com
  • Senha: SuaSenhaDoOffice
  • Conexão Segura: Desmarcado (O Outlook usa STARTTLS na porta 587, que o sistema negocia automaticamente)
  • Email de Envio (From): Deve ser idêntico ao usuário autenticado.

3. Servidores Genéricos (cPanel, Postfix, etc.)

Consulte seu provedor de hospedagem. Geralmente:

  • Host: mail.seudominio.com
  • Porta: 465 (SSL) ou 587 (TLS)
  • Seguro: Marque para 465, desmarque para 587.

Solução de Problemas

  • Erro de Autenticação: Verifique se a senha está correta ou se precisa de uma "Senha de App".
  • Timeout: Verifique se o firewall da rede não bloqueia as portas 465 ou 587.
  • Logs: O sistema grava tentativas de envio em um arquivo de log no servidor (email.log) para auditoria.
